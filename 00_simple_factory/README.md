## Go 简单工厂模式

```bash

关键点：
  由于 Go 本身是没有构造函数的，采用 NewName 的方式创建对象/接口，当它返回的是接口的时候，其实就是简单工厂模式
  实现简单 有较多if else分支，适用于改动不频繁的方法





如何判断是否使用工厂模式：
  封装变化：
    创建逻辑有可能变化，封装成工厂类后，创建逻辑的变更对调用者透明
  代码复用：
    创建代码抽离到独立的工厂类之后可以复用
  隔离复杂性： 
    封装复杂的创建逻辑，调用者无需了解如何创建对象
  控制复杂度
    将创建代码抽离出来，让园本的函数或类职责更单一，代码简洁
```
